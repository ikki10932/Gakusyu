# Document Review Advisor Skill

このスキルは、提供された資料をレビューし、将来の担当者が3年後でも即座に理解・再現できるような資料へと改善するための具体的なアドバイスを生成するために使用されます。

## 目的

単なる内容の要約ではなく、「将来の担当者が迷わず再現可能か」を基準として、資料の構成、レイアウト、必須記述事項を最適化します。

## 解析およびアドバイスの観点

### 1. 長期的保守性
- 3〜5年後でも社内開発メンバーが理解可能な内容か。
- 属人的な判断を排除し、事実に基づいた記述になっているか。

### 2. 再現性と理解容易性
- 設計意図や前提条件、依存関係が明確に示されているか。
- 専門用語が適切に定義され、新任担当者が単独で理解できるレベルか。

## 出力内容（厳守）

レビュー結果は以下の項目を含む構造で提供されます。

1.  **タイトル・概要**: 資料の冒頭に記述すべき2〜3行の簡潔な説明。
2.  **改善の方向性**: 資料をどのような方針で整理すべきか。
3.  **問題点**: 長期閲覧・保守の観点から見た現在の資料の弱点。
4.  **推奨構成**: 資料の目次・章立ての提案。
5.  **必須記載項目**: 各構成セクションにおいて、必ず記述すべき具体的な内容。
6.  **補強提案**: 将来の担当者の理解を助けるための追加提案。
7.  **欠落情報一覧**: 現時点で不足している、あるいは「不明」な情報のリスト。
8.  **エグゼクティブサマリー**: 上記内容の最短まとめ（箇条書き）。

## 保存および制約

- **保存ルール**: 回答生成後、必ず `./output` ディレクトリに `YYYY-MM-DD_内容キーワード.md` 形式で保存します。
- **制約事項**:
    - 推測禁止。不明点は「不明」と明示。
    - 曖昧な表現や主観的な感想を排除。
    - 再現性を最優先事項とする。
