# Router Configuration I/O Trace Analysis Skill

このスキルは、ルータの設定ファイル（.dat）と、それを処理する C++ ソースコード間のデータフローを厳密に解析するために使用されます。

## 目的

以下の要素を事実に基づいて特定し、構造化されたレポートを生成します。

1.  **どの関数が**: 設定ファイルの読み書きを担当しているか。
2.  **どの領域を**: .dat ファイル内の具体的なデータ位置。
3.  **どのメンバーへ**: 解析対象構造体（`SDkRouter` 等）のどの変数に関連付けられているか。
4.  **I/O種別**: 読み込み（Read）か書き込み（Write）か。

## 解析の原則

- **事実ベース**: 必ずソースコードのファイル名と行番号を根拠として提示します。
- **推測の排除**: コードから直接確認できない事項は「不明」とし、推論が必要な場合は必ず「推論」と明記します。
- **実コード優先**: コメントではなく、実際の実行コード（fread/fwrite, ifstream/ofstream等）を根拠とします。

## 標準解析フロー

### 1. 構造体定義の完全展開
解析対象の構造体（例：`SDkRouter`）の全メンバー、型、配列サイズ、ネスト構造を抽出し、`work/struct_definition.md` にまとめます。

### 2. I/O 関数の特定
ファイル操作（fread, fwrite, read, write, ifstream, ofstream）や、入出力に関連するキーワード（Load, Save, Read, Write）を含む関数をリストアップし、`work/io_function_list.md` に出力します。

### 3. メンバー単位のトレース（重要）
各構造体メンバーについて、以下の情報を整理し `work/member_trace.md` に保存します。
- 書き込み/読み込み関数
- ファイル名と行番号
- アクセス方式（一括、個別、バッファ経由）

### 4. 実ファイル（.dat）照合
実際の `.dat` ファイルの内容とコードの挙動を照合し、データの整合性を確認します。

## 最終成果物
`output/YYYY-MM-DD_dat_io_report.md` を生成します。レポートには以下を含めます。
- I/O 関数一覧（根拠行付き）
- メンバー別対応表（読込/書込関数、保存形式）
- .dat 構造の判定（固定長、個別、ヘッダ有無など）
- 重要関数の影響分析（特定の関数が何を書き換えるか）
