# MFC Legacy Code Analysis Skill

このスキルは、保守性の低下した MFC レガシープロジェクトを解析し、コードの属人性を排除して将来のリファクタリングや機能追加を容易にするために使用されます。

## 解析対象と目的

- **クラス構造**: 継承関係、メンバー変数、主要な責務の明確化。
- **Doc/View 責務**: ビジネスロジックが View に混入していないか、UI 依存が適切に分離されているかを評価。
- **メモリ管理**: `new`/`delete` の対応、ポインタの所有権、リークリスクの特定。
- **依存関係**: Include 構造、グローバル変数の使用、循環参照の有無。

## 重点解析項目

### 1. メモリ所有権（最重要）
各ポインタ変数について、「誰が生成し、誰が解放するのか」を追跡します。不明な場合は「所有権不明」としてリスク登録します。

### 2. メッセージマップと巨大関数
`BEGIN_MESSAGE_MAP` を解析し、UI イベントハンドラ内のロジック肥大化（100行超）を検出・警告します。

### 3. 肥大化診断
- 1,000行を超えるファイル。
- UI と業務ロジックの混在。
- 複数の責務を持つ神クラス（God Class）の特定。

## 生成される資料

1.  **クラス一覧**: 種別、責務、検出された問題点。
2.  **所有権マップ**: ポインタごとの生成・解放経路。
3.  **依存関係要約**: モジュール間の結合度。
4.  **リファクタリング優先順位**: 修正による効果とリスクに基づいた優先度評価。

## 品質基準

解析結果は、「新規担当者がそのコードベースの構造を半日で理解できるレベル」の明文化を目指します。
