# Excel List Layout Skill

このスキルは、データを読みやすく、そのまま業務資料として利用可能な形式で Excel 出力するために使用されます。

## 基本原則

- **データダンプの禁止**: 単純な CSV 形式の出力ではなく、適切なヘッダーや枠線、書式を設定します。
- **レイアウト設計**: 列幅の固定、セルの配置（上下中央、ヘッダーの中央寄せ）を徹底します。
- **データ型の維持**: 日付や数値が文字列として出力されないよう、適切な型を指定します。

## 設計ルール

### シート構成
- シート名は具体的な内容を示す名称にします（`Sheet1` は禁止）。
- 1行目にタイトル（任意）、2～3行目に内容の説明文(一行最大80文字)、4行目にヘッダー、5行目以降にデータを配置する標準構成を推奨します。

### セル書式
- **ヘッダー**: 太字、背景色、中央寄せ。
- **データ領域**: 上下中央配置。
- **列幅**: ID系(15), 名称系(25), 数値系(12) など、内容に応じて最適化します（自動調整に頼らない）。

## 出力メタ定義
Excel 出力要求時には、以下の仕様をあらかじめ定義します。

1.  **列仕様**: 列名、型（文字列/整数/日付等）、表示形式、列幅。
2.  **ソート条件**: デフォルトの並び順。
3.  **集計仕様**: 合計や平均が必要な場合の対象列と位置（最下行）。

## 禁止事項
- セルの結合（データ領域内）。
- 空白行・空白シートの放置。
- 型情報の欠落。
